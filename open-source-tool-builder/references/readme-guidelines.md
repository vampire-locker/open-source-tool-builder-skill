# README Guidelines

Write README files for real users.

## Recommended Order

1. Project name
2. Language switch links, if bilingual
3. One-sentence summary
4. What it does
5. Requirements
6. Install
7. Quick start
8. Common commands
9. Configuration
10. Uninstall
11. Troubleshooting
12. Build and test
13. Release process
14. License

## Installation Guidance

Rank install methods by user friendliness:

1. Official binary/package download
2. Stable package manager command, only if published
3. Language-native install command, only if valid
4. Build from source

Do not present future package managers as current install paths.

## Anti-Patterns

- Leaving fake owner names, fake repository names, placeholder markers, future promises, or fake commands in public docs.
- Explaining implementation before user value.
- Mixing developer build instructions into the first install path.
- Omitting uninstall or cleanup for tools that install background tasks.
- Hiding platform permission requirements until troubleshooting.
